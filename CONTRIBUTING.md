# Contributing to `skelFMM`

Thanks for your interest in contributing.

## Pull requests

Pull requests are welcome.

Please include:

- a short summary of what changed
- any relevant numerical, API, or performance impact
- the tests or examples you ran

If a change is still exploratory, that is fine; a short note in the pull
request description is helpful.

## Questions and bug reports

Please use GitHub Issues for bug reports, feature requests, or questions about
the package.
